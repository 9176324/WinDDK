GFX

NOTE: This sample will only build for 32-bit Microsoft® Windows® XP or later platforms.

SUMMARY

The GFX sample code is divided into two projects, which reside in the GFXProp and driver subdirectories. The following table describes the contents of these two subdirectories.

Subdirectory

Description

GFXProp

This sample shows how to write a property dialog DLL that gets loaded by the multimedia control panel when a user selects the Properties button on the Effects tab of the Advanced Audio Properties dialog. To open the properties dialog, click on the Advanced button of the audio tab of the multimedia control panel. With this property dialog, the user can change all the configurable settings of the GFX filter. This sample requires the audio GFX filter sample to be installed.

driver

This sample is the actual audio GFX filter.

The INF file in this directory installs both samples. The samples in the subdirectories driver and GFXProp communicate with each other. The inc subdirectory contains cross-project header file, which defines the GUIDS that identify properties and names.

GUID DEPENDENCIES BETWEEN THE SAMPLES

GUIDs are typically used to facilitiate communication between independent software modules. Successful communication requires the modules to agree on the meaning of a set of shared GUID values. Thus, if the programmer modifies a shared GUID value in one module, the programmer must modify the corresponding GUIDs in the other modules as well.

In the GFX samples, there are several GUID dependencies between the projects in the driver and GFXProp subdirectories. To adapt either the driver or GFXProp sample for your needs, avoid modifying the project in the existing subdirectory. Instead, create a new project directory and use the sample code as a starting point for the new project. After creating the new project, determine to which of the following categories each GUID in the project belongs:

·         GUIDs that need to be unique for each project

·         GUIDs that need to be known between projects

·         GUIDs that are predefined

This approach eases the difficulty of keeping track of all the GUIDs.

GUIDS THAT NEED TO BE CUSTOMIZED

You will need to modify the following GUIDs when you adapt the sample code for your new project:

·         The GUID assigned to GFX.DeviceDesc in the INF file. The same GUID appears again in the INF file with the name GFX.DeviceId.

·         The GUID added to registry pathname Gfx\Userinterface\CLSID in the INF file. The same GUID appears in the section GFXPROPLib in the file GFXprop.idl.

·         The GUID with the name GFXGUID.ChannelSwap in the INF file. The same GUID is used as the GFX node name in the header file msgfx.h.

·         The GUID named GFX.Ref in the INF file. The same GUID is used as the GFX filter name in the header file msgfx.h.

GUIDS THAT NEED TO BE THE SAME ACROSS PROJECTS

The following GUIDs need to have the same values across projects:

·         The GUID added to registry pathname Gfx\Userinterface\CLSID (GUID for GFX property page object/library) in the INF file and used in section GFXPROPLib of file GFXprop.idl

·         The GUID named GFXGUID.ChannelSwap in the INF file and named GFXSAMPLE_NODENAME_CHANNEL_SWAP in file msgfx.h

·         The GUID named GFX.Ref in the INF file and named KSNAME_MsGfxSample in file msgfx.h

If you change the GUID in one project, then change the corresponding GUID in the other project as well. For example, a property request can succeed only if the two projects agree on the GUID that identifies the property set.

GUIDS THE VC6++ WIZARD TAKES CARE OF WHEN YOU CREATE A NEW ATL PROJECT

There are also several GUIDs that the ATL Wizard generates when you create a new ATL project. If you are going to modify the GFXProp sample, then you will need to change the following GUIDs:

·         In file GFXProperty.rgs, modify the CLSID for the GFXProperty class (3 instances); the same GUID is in file GFXProp.idl under "GFXPropPages".

·         In file GFXPropPages.rgs, modify the CLSID for the GFXPropPages class (3 instances); the same GUID is in file GFXProp.idl under "GFXPROPLib" and in the INF file under Gfx\Userinterface\CLSID.

·         In file GFXPropPages.rgs, modify the TypeLib GUID; this GUID is also used in file GFXProp.idl.

HOW TO INSTALL THE SAMPLE

To install the sample, right click on the INF file and choose the Install option. The system will then copy the INF file to the "Program Files\DDK" directory and, after a few seconds, execute the INF file from that directory to install both the GFX filter and the property DLL.

Open the Advanced Audio Properties of your USB speakers and click the Effects tab:
    Control Panel -> Sounds and Audio Devices -> Audio tab -> Select USB speakers -> Advanced button -> Effects tab
If you have Microsoft DSS 80 USB speakers, then you will see a combo box where you can select and apply the GFX filter. If you have a different type of USB speakers, then modify the INF file before installation by replacing the Plug and Play device-identification string of the Microsoft DSS 80 USB speakers with the Plug and Play device-identification string of the USB speakers that you have.

NOTE: If you need to change the device ID for your USB speakers, do so before installing the GFX filter. You will be able to install the GFX filter only once on a given test machine. You will not be able to uninstall the GFX filter once it has been installed.

ADDITIONAL INFORMATION TO THE INF FILE

Although the INF file will try to install the audio GFX sample together with the property dialog on Windows 2000 (and later), the sample will work only on Microsoft Windows XP (and later).
