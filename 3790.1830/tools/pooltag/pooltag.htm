PoolTag Utility

 

SUMMARY

PoolTag is a GUI-based version of the venerable PoolMon utility that has long been provided in the DDK.  PoolTag shows the amount of paged and non-paged pool used, by the pool tag that was provided at the time of allocation.  The number of successful allocations, frees, and the total number pool allocations that have not yet been freed are all reported for each pool tag.  Filters, including trailing wildcard characters, can be set to limit the information displayed to a sub-set of pool tags.

PoolTag is useful for determining paged and non-paged pool usage, finding pool leaks, and identifying hot spots in pool usage by kernel mode components.

Current Status

PoolTag is a Win32 executable that runs entirely in user-mode.  It has been well tested in a variety of environments.  The current version of PoolTag is V2.2. This version of PoolTag includes a small fix in PoolTag support for Windows XP SP1.

PoolTag was written by OSR Open Systems Resources, Inc. (http://www.osr.com) and is distributed in the Windows DDK under license.

Running the Tool

When run, PoolTag first checks to see if pool tagging is enabled on the system.  If it is not, it offers to enable system pool tagging for the next reboot.

The information displayed by PoolTag is self-explanatory.

Clicking on the “Pool Tags” menu item, and selecting “Filters and Update Interval…” allows you to specify a set of filters, optionally with a trailing wildcard “*” character, to limit the information displayed.  Specify one filter per line.  Filters are normally case sensitive, unless the “Filters are Case Insensitive” check box is checked.  From this menu item you can also change the information update interval, or save or restore a previously saved PoolTag configuration.  The saved configuration includes the list of filters, case sensitivity flag, and update interval.

The “Filters and Update Interval” dialog may also be used to establish a “baseline snapshot” of pool usage.  When this is done, PoolTag shows only the changes from that baseline.  This can be very useful for identifying pool leaks as well as frequent allocations/deallocations.

PoolTag’s display information (including a list of configured filters) can also be saved to disk in a text file by clicking on the “File” menu item, and selecting “Save As…”.  You will be prompted for the specification of the file to which to save the pool tag information.

Limitations

Though expected to work, PoolTag has not been thoroughly tested in IA64 environments.

PoolTag supports or Microsoft Windows® NT, Windows 2000, Windows XP and Windows Server 2003.  It does support Microsoft Windows 95, Microsoft Windows 98, Microsoft Windows Millennium.
