Overview

The WDK provides a platform on which to develop, compile, and test drivers for Windows. To obtain a logo for your driver, use the Windows Logo Kit (WLK), which is available separately. The WDK provides driver‑specific headers, libraries, sources, tools, and documentation for developing drivers that are compatible with the following operating systems:

·         Microsoft Windows® 7

·         Microsoft Windows Vista®

·         Microsoft Windows XP

·         Microsoft Windows Server® 2008 R2

·         Microsoft Windows Server® 2008

·         Microsoft Windows Server® 2003

The WDK build environments do not support the development of drivers for the following operating systems:

·         Microsoft Windows Millennium Edition

·         Microsoft Windows 98

·         Microsoft Windows 95

·         Microsoft Windows NT® 4.0

·         Microsoft Windows 2000

 

You will need a DVD‑ROM and 4.7 GB of free hard disk space to install the full WDK and compile all the samples.

You can install the WDK on the following operating systems:

·         Windows 7

·         Windows Server 2008

·         Windows Server 2008 (R2)

·         Windows Vista (all versions)

·         Windows Server 2003 with Service Pack 1 (SP1) or later

·         Windows XP with Service Pack 3 (SP3) or later

 

In order to install the WDK on Windows XP Professional x64 Edition, you must manually install .NET Framework 2.0 (if not already installed) using the package available on the media at \Redist\Net2.0\x64\Netfx64.exe before launching the WDK install.

 

Install the Windows Driver Kit

Use KitSetup.exe to install the WDK Development Environment, WDK Documentation, the Kernel Debugger, Device Simulation Framework, and Windows Device Testing Framework. All prerequisite software, such as .NET version 2.0, and Windows Document Explorer will be installed as well.

By using KitSetup.exe, you can select the components you want to install by selecting or clearing check boxes in the Features area of the KitSetup dialog box. By default, all check boxes are clear, which indicates that none of the WDK components has been installed. When you select a feature, status for that feature is displayed in Feature Status area of the dialog box.

As you select check boxes, you can see the components you have selected in the Pending setup tasks area of the dialog box.

Device Simulation Framework

The Device Simulation Framework for USB Devices provides an architecture and programming framework to simulate USB devices in software.

Only one version of DSF can be installed on Windows. We recommend that you do not install DSF on the same machine as the rest of your development environment. If DSF is already installed, the installed version of DSF must be removed before proceeding (see Uninstalling DSF on MSDN). DSF installs kernel-mode drivers on Windows and could potentially destabilize the system. KitSetup.exe will display a warning message if your system is in danger. For more information about DSF, see Device Simulation Framework for USB Devices on MSDN.

Installation Location Dialog Box

The Select Kit Install Location dialog box displays the default installation path for the WDK. You can click cancel and return to the KitSetup dialog box at any time. Click Browse to choose a new installation path for the WDK. Click Default to restore the default installation path.

After you click OK, the installation continues to the End-User-License-Agreement dialog box.

Verify the WDK Build Environments Installed Correctly

Verify that the installation succeeded.

Note  The default installation path is x:\WinDDK\<BldNum>. The icons are located at Programs\Windows Driver Kits\Build Number\.

To verify that the WDK Build Environments installed successfully

1.    Click Start.

2.    Point to All Programs > Windows Driver Kits > WDK build# > Build Environments > Windows 7.

3.    Click x86 Checked Build Environment.

4.    At the command prompt, type build -cZ.

This build command compiles and links the complete set of installed sources. The build process can take more than 30 minutes to complete, depending on the WDK components that you installed and the system that you are installing on.

The build command should complete with no errors and no warnings.

The WDK installation process copies the release notes file from the installation media to the root directory of the WDK installation.

 

Manually Install Debugger(s) (Optional)

The Debugging Tools for Windows package features WinDbg, a powerful debugger with a graphical interface that can debug both user‑mode and kernel‑mode code. This package also includes the debuggers KD, NTSD, CDB, and many additional tools. The documentation in this package describes the use of these debuggers and includes tips for user‑mode and kernel‑mode debugging.

Debugging Tools for Windows is available in 32‑bit and 64‑bit versions. The 32‑bit version is appropriate for most users. However, if you want to debug an application on a processor that is based on 64 bits, you should select the 64‑bit version. The 64‑bit version can be installed only on 64‑bit versions of the Windows operating system.

To obtain the most current version of Debugging Tools for Windows, see Microsoft Debugging Tools.

 

KitSetup.exe can automatically install the correct debugger for your platform if you select the Debugging Tools for Windows check box during installation.

Note  This method installs the debugger to a folder called Debugger, which is located under the WDK install path that you select during installation.

 

If you cannot install the most recent version of the Debugging Tools for Windows from the Internet, you can manually install the debugger that is appropriate for your platform:

·         For x86‑based platforms, run Setup_x86.exe from the Debuggers folder of the installation media.

·         For IA64‑based platforms, run Setup_ia64.exe from the Debuggers folder of the installation media.

·         For AMD86‑based platforms, run Setup_amd64.exe from the Debuggers folder of the installation media.

Note  This method installs the debugger to the default install location of the debugger, which is under Program Files.

 

Install Windows Device Testing Framework (WDTF)

The Windows Device Testing Framework (WDTF) enables you to create, manage, reuse, and extend device‑centric, scenario‑based automated tests. We recommend you install this component along with the WDK to use existing tests or create new tests. This component is also required on test computers to use WDTF‑based tests.

If WDTF is already installed, the existing version of WDTF will be replaced by KitSetup.exe.

For more information about WDTF, see Windows Device Testing Framework on MSDN.
