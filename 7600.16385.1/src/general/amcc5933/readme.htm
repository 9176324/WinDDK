AMCC5953

SUMMARY

This sample demonstrates how to write driver for a generic PCI device using Windows Driver Framework. The target hardware for this driver is AMCC PCI Matchmaker Developer’s kit (S5935DK1) board. The product kit and the hardware specification are available at https://www.amcc.com.

 

The kit contains a PCI development board and an ISA card that connects to the development board via a ribbon cable. The ISA card allows you to access the development board to provide the software side simulation of add-on device.

 

Following table gives a typical device and driver scenario and also lists the driver framework interfaces demonstrated in this sample:

 

Scenario

Features Demonstrated

The device is a PCI device with port, memory, interrupt and DMA resources. Device can be stopped and started at run-time and also supports low power state.

 

The driver is capable of doing concurrent single read and write operations at any time.

·         Handling PnP & Power Events

·         Registering Device Interface

·         Hardware resource mapping: Port, Memory & Interrupt

·         DMA Interfaces

·         Serialized Default Queue for Write requests

·         Serialized custom Queue for Read requests

·         Handling Interrupt & DPC

·         Event Tracing

 

This sample also demonstrates how to use a single driver binary to control two different types (PCI & ISA) of devices.

This sample works on Windows® 2000 and later operating systems.

This sample driver is a minimal driver meant to demonstrate the usage of the Windows Driver Framework. It is not intended for use in a production environment.

BUILDING THE SAMPLE

To build the sample drivers, you must first set up the DDK environment on your host machine. The “Installation and Release Notes” in the Windows 2000/XP/Server 2003/Vista/Server 2008/Windows 7 DDK has a complete description on how to do this.

Change to the directory containing the device source code, such as CD src\general\AMCC5953.

Run build -ceZ, or use the macro BLD. This command invokes the Microsoft make routines to build the components. If the build succeeds, you will find the driver, binaries under the respective subdirectories of the source file depending on the build environment. If it fails you can find errors and warnings in the buildxxx.err and buildxxx.err respectively, where xxx is either chk or fre depending on the build environment.  Copy the KMDF coinstaller (wdfcoinstallerMMmmm.dll), sys, INF and test binaries to a floppy or to a directory of a test machine.

INSTALLATION

Plug in the PCI development board and the ISA card and start your system. When you login into Administrator account, the system will welcome you with a ‘Found New Hardware Dialog’ for “Other Bridge Devices”. Follow the instruction given below to install the driver for the PCI device.

On Windows 7 and later operating systems:

1.  Launch “Device Manager” by executing command devmgmt.msc in a command window, or from “Hardware and Sound” program group in Control Panel.

2.  Select “PCI Device” device from “Other Devices” category and click “Update Driver Software…” from right-click menu.

3.  Select Browse my computer for software and provide the location of the driver files.

4.  Select “Install this driver software anyway” when the “Windows Security” dialog box appears.

5.  The system will complete the installation, load the driver and start the device.

 

On Windows Vista and Windows Server 2008:

1.    In the “Found New Hardware” dialog box, select “Locate and install driver software (recommended)”.

2.    Select “Don't search online”.

3.    Select “I don't have the disc. Show me other options”.

4.    Select “Browse my computer for driver software (advanced)”.

5.    Enter the location of the driver files, and select “Install this driver software anyway” when the “Windows Security” dialog box appears.

6.    The system will complete the installation, load the driver and start the device. 

 

On Windows XP and Windows Server 2003:

1.    Select 'Install from a list or specific location (Advanced)”, then click Next.

2.    Select ‘Add a new hardware device’ from the list, then click Next.

3.    Select ‘Select for the best drivers in these locations' and then select, “Include this location in the search path” check box and specify the media or the directory that contains the driver package and then click Next.

4.    You will get a Security Alert about installing unsigned driver. Ignore that and continue the installation by clicking on “Yes”.

5.    The system will complete the installation, load the driver and start the device.

 

On Windows 2000:

1.    Select “Display list of known devices so that I can a specific device”, and then click Next.

2.    Select the “Other Class” in the Hardware type, and click Next.

3.    Click on the ‘Have Disk’ and specify the media or the directory that contains the driver files, and then click Next.

4.    You will get a warning message about installing unsigned driver. Ignore that and continue the installation.

 

The Driver for ISA device has to be installed manually because it is a non plug & play ISA device.

 

On Windows 7 and later operating systems:

1.    Run “hdwwiz.exe” in a command window or in the “Search programs and files” of the “Start” menu.

2.    At the "Welcome..." click on "Next".

3.    At the "The wizard can help..." panel, select "Install...manually..." and then click "Next".

4.    At the "From the list below..." panel, select "Show All Devices" and then click "Next".

5.    At the "Select the device driver..." panel, just click on the "Have Disk..." button.

6.    At the "Install From Disk" panel, click the "Browse..." button. From the Locate File panel specify the directory location where the AMCC install files can be found: AMCC5933.INF and AMCC5933.SYS. Click "OK" to return to the "Select the device driver..." panel.

7.    In the "Model" list box, you should see a listing for "AMCC S5993 Development Kit ISA Adapter".  Select this entry and then click "Next".

8.    At the "The wizard is ready..." panel, click "Next".

9.    At the final "Completing the Add Hardware Wizard" panel, click "Finish".

 

On Windows Vista and Windows 2008:

1.    Start the control panel and bring up the "Add Hardware" wizard.

2.    At the "Welcome..." click on "Next"

3.    At the "The wizard can help..." panel, select "Install...manually..." and then click "Next".

4.    At the "From the list below..." panel, select "Show All Devices" and then click "Next".

5.    At the "Select the device driver..." panel, just click on the "Have Disk..." button.

6.    At the "Install From Disk" panel, click the "Browse..." button. From the Locate File panel specify the directory location where the AMCC install files can be found: AMCC5933.INF and AMCC5933.SYS. Click "OK" to return to the "Select the device driver..." panel.

7.    In the "Model" list box, you should see a listing for "AMCC S5993 Development Kit ISA Adapter".  Select this entry and then click "Next".

8.    At the "The wizard is ready..." panel, click "Next".

On Windows XP and Windows 2003:

1.    Start the control panel and bring up the "Add Hardware" wizard.

2.    At the "Welcome..." click on "Next"  (this will cause scan of device)

3.    At the "Is the Hardware connected" panel, select "Yes,..." and then click "Next".

4.    At the "The following hardware is already installed..." panel, select "Add a new hardware device" and click "Next"

To install the ISA driver on Windows 2000:

1.    Double-click the Add New Hardware Wizard applet in Control Panel.

2.    Click Next.

3.    Select Add a new device.

4.    Select No, I Want to Select the Hardware from a list.

5.    Select Other Devices and then click Next.

6.    Click Have Disk and point to the directory that contains sys and INF file.

 

NOTE: Please do not load the PCI driver on a checked build Windows 2000 because WdfInterruptAcquireLock causes a benign assert in the kernel. Only way to avoid the assert would be to use KeSynchronizeExecution function instead of WdfInterruptAcquire/ReleaseLock to synchronize execution with the ISR().

TESTING

To test the driver, run the TxTest.exe and RxTest.exe. Please make sure both ISA and PCI devices are installed and started before you run the application.
