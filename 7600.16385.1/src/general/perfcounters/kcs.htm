Kernel Counter Sample (Kcs)

SUMMARY

kcs is a sample to demonstrate the use of kernel mode perflib API. The driver does not control any hardware; it simple provides counters. The code contains comments to explain what each function does.

USING THE SAMPLE

To test the Kcs counter provider, use the following procedure.

  1. Build the kcs sample driver by using the -cZg switch. For more information, see "Building a Sample Driver" in the DDK.

  2. Copy kcs.sys and kcs.man files in %windir%\system32 dir and kcs.sys.mui in %windir%\system32\en-US dir.

  3. Sign the sample driver. For details about how to sign the driver, see "SIGNNING THE SAMPLE DRIVER" section.

  1. Install the counters.

lodctr.exe /m:kcs.man

  1. Create a service to control the driver.

sc create kcs binpath= %windir%\system32\kcs.sys type= kernel start= demand

  1. Start the service

net start kcs

  1. Run perfmon and add Geometric Wave and Trignometric Wave Counter Sets

perfmon.exe

  1. To stop the service, run

net stop kcs

  1. To un-install the counters, run

unlodctr.exe /m:kcs.man

SIGNNING THE SAMPLE DRIVER

To sign the simple driver (kcs.sys), use the following procedure.

  1. Open an elevated command window.
    • Click the Start button, point to All Programs, Windows Driver Kits, and WDK, and click Build Environments\Windows Win7.
    • Right-click x64 Free Build Environment (or other environment which you compile the code) and select Run As Administrator from the shortcut menu.
    • Go to the directory which contain the kcs.sys and kcs.pdb.(ie. C:\WinDDK\7060.0\src\general\perfcounters\kcs\objfre_win7_amd64\amd64).
  1. Create a Test Certificate by Using MakeCert.

    2. makecert -r -pe -ss PrivateCertStore -n CN=Contoso.com(Test) kcs.cer

  1. Create a Catalog File for Test-Signing.
    • Create a file named kcs.cdf, and Open Notepad and copy the text from the following to the file.

      • [CatalogHeader]

      Name=kcs.cat

      PublicVersion=0x0000001

      EncodingType=0x00010001

      CATATTR1=0x10010001:OSAttr:2:6.0

      [CatalogFiles]

      <hash>File1=kcs.pdb

      <hash>File2=kcs.sys

    • Run MakeCat on kcs.cdf.

      • makecat -v kcs.cdf

  1. Install the Test Certificate in the Trusted Root Certification Authorities Certificate Store.

    2. certmgr.exe /add kcs.cer /s /r localMachine root

  1. Test-Sign a Driver Image File by Using an Embedded Signature.

    2. Signtool sign /v /s PrivateCertStore /n Contoso.com(Test) /t http://timestamp.verisign.com/scripts/timestamp.dll kcs.sys

  1. Use SignTool to verify the test-signature on kcs.sys.

    2. Signtool verify /pa /v kcs.sys

 

LIMITATIONS

This sample driver should not be used in a production environment.

 

Kcs is designed for Windows 7 and later versions of Windows.
