Introduction

NICStats is a simple command line based tool that provides information on network adaptors installed in a server. The tool queries OIDs and counters at the NDIS level and provides useful information on a Network Interface Cards (NIC) capabilities and configuration details.

NICStats can be used to gather the following information

·         NIC capabilities

·         NIC driver information

·         RSS capabilities

·         RSS parameters (Indirection table, etc.)

·         Power management information

·         NIC specific performance counters and event cycles

This information applies for the following operating systems:
                Windows Server® 2008 R2
                Windows 7

How to use NICStats

The application included in this package is named NICStats.exe.

To run NICStats, open up a command prompt with administrator privileges. Use commands listed in the NICStats examples section to verify that the tool works fine.

NICStats does not require any other special configurations.

NICStats Examples

1.       Enumerate all physical adaptors in the machine

nicstats -a Enum -t AllPhysical

2.       Get hardware info for the adaptor with ifIndex of 12

nicstats -t ifindex -i 12 -a GetHardwareInfo

3.       Get RSS capabilities for the adaptor with ifIndex of 12

nicstats -t ifIndex -i 12 -a GetRSSCaps

4.       Get RSS parameters for the adaptor with ifIndex of 12

nicstats -t ifIndex -i 12 -a GetRSSParams

5.       Get power management info for the adaptor with ifIndex of 12

nicstats t ifIndex -i 12 -a GetPowerInfo

NICStats Reference

This section provides detailed information of all the command line arguments supported by NICStats.

-a <action>

Description

Performs action selected option (from list below)-

1.       Enum                                            - Lists all adaptors installed in the system

2.       GetHardwareInfo                    - Provides hardware information for the selected interface (Should be used with -i option)

3.       GetRSSCaps                               - Returns RSS capabilities of the interface (RSS revision, number of queues,...)

4.       GetRSSParams                          - Returns RSS parameters of the interface (Hash function, hash fields, indirection table)

5.       CalcRssCpu                                 - Returns the RSS target processor for an interface given the source and destination IP address and source and destination TCP port numbers

6.       ResetPerfCounters                                 - Resets all per processor performance coutners to zero

7.       StartPerfCounters                   - Start per processor performance counters. (Use -c and -y options to specify coutners.)

8.       StopPerfCounters                   - Stop per processor performance counters

9.       GetPerfCounters                     - Displays per processor perfromance counters

10.   GetPowerInfo                           - Returns power management capabilities of the selected interface

Default Value

Enum

-c <event count tracking flags>

Description

DWORD describing counters selected for tracking. Used with -a StartPerfCounters

Bitmap specifying which event to keep track of:

·                     DPC count                                                           0x00000001

·                     ISR count                                                             0x00000002

·                     Receive_Indications                                       0x00000004

·                     Return_NBL calls                                              0x00000008

·                     Received NBLs                                                  0x00000010

·                     Returned NBLs                                                  0x00000020

·                     Send calls                                                            0x00000040

·                     Send_Complete calls                                      0x00000080

·                     Sent NBs                                                              0x00000100

·                     Sent_Complete NBs                                       0x00000200

·                     HAL build_SG calls                                           0x00000400

·                     Indirection table changes                             0x00000800

·                     Low resource recv indication calls             0x00001000

·                     Low resource recv indication NBLs           0x00002000

·                     Queue DPC on non-current processor    0x00004000

 

DO NOT specify 0x before the mask.

-d

Description

Specifies destination related info.

 IP=<IP destination address> is the IP address of the LOCAL host

TCP=<TCP destination port> is TCP port number of the LOCAL application

-i <interface ID value>

Description

Interface identifier.

The interface identifier value is interpreted based on the interface type specified in the t option. If this option if the Action needs an interface, an interface is not specified, and there is only one physical adaptor in the system, it will be picked as the default interface.

-o <output file name>

Description

Stores the output in the specified file name.

Default Value

stderr

-s

Description

Specifies source related info.

IP=<IP source address> is the IP address of the REMOTE host

TCP=<TCP source port> is the TCP port number of the REMOTE application

-t <Interface ID Type>

Description

Specifies type of interface to be considered

1.       ifIndex                         - Interface index

2.       ifName                         - NetLUID

3.       ifAlias                           - Conenction name

4.       ifDesc                           - Friendly name, DeviceName (using \Device\{GUID})

5.       ALL                                 - All adaptors

6.       AllPhysical                  - All physical adaptors

7.       EnumIndex                 - Use the index returned back by Enumeration

Default Value

AllPhysical for -a Enum

ifIndex for the rest

-x <get cost>

Description

-y <event cycle count tracking flags>

Description

DWORD specifying which which event cycle counts should to be recorded.

Bitmap specifying which event cycles to keep track of:

·         Total DPC cycles                                        0x00000001

·         ISR cycles                                                     0x00000002

·         NDIS receive_indication cycles           0x00000004

·         Stack receive_indication cycles          0x00000008

·         Return_NBL NDIS cycles                       0x00000010

·         Return_NBL miniport cycles                                0x00000020

·         NDIS send cycles                                      0x00000040

·         Miniport send cycles                              0x00000080

·         NDIS send_complete cycles                                0x00000100

·         Stack send_complete cycles                               0x00000200

·         HAL build_SG cycles                                                0x00000400

·         Indirection table change cycles          0x00000800
