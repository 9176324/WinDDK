DMusUART

SUMMARY

This is the DMusUART sample driver, which is a DMus miniport driver. It provides applications that use DirectMusic® or the Microsoft® Windows® multimedia midiInXxx and midiOutXxx functions with an interface to the MPU-401 chip.

The DMus port driver, which is built into PortCls.sys, performs the generic KS filter functions for this miniport driver and connects it to the rest of the system. The sample driver handles the following:

  • One input stream visible to either DirectMusic or the midiInXxx functions
  • One output stream visible to the midiOutXxx functions
  • Multiple output streams visible to DirectMusic

The sample driver compiles in both 32- and 64-bit environments and is compatible with both the x86 and Alpha platforms.

The PortCls system driver automatically handles Plug and Play and power management on behalf of the miniport driver.

BUILDING THE SAMPLE

To build the sample, execute the build command in the DDK sample directory src\wdm\audio\dmusuart. If the build succeeds, it produces the dmusuart.lib library module, which can be linked into an audio adapter driver.

RESOURCES

The DMusUART sample driver is identical to the system-supplied DMusUART miniport driver in PortCls.sys. The source code for this sample can serve as a starting point for developers who need to write simple DMus miniport drivers. (For more complex DirectMusic devices, including those that have DLS functionality, please instead refer to the DirectMusic kernel-mode software synth sample in the ddksynth sample directory.) The DMusUART miniport driver interfaces to the DMus port driver in PortCls.sys.

In addition to the DMus port driver, PortCls.sys contains a MIDI port driver that provides basic MIDI functions but lacks support for the advanced MIDI features in DirectMusic. Unlike the DMus port driver, the MIDI port driver is available in Windows 98 Gold. The DMus port driver provides higher functionality than the MIDI port driver, so if you are creating your own miniport driver, consider interfacing to DMus instead of MIDI. A DMus miniport driver has the following advantages:

  • A DMus miniport driver is visible to the DirectMusic API, unlike a MIDI miniport driver, which is not.
  • A DMus miniport driver is visible to the older, Windows multimedia midiInXxx and midiOutXxx functions if the driver includes a legacy pin descriptor.

In short, DMus miniport drivers have the benefits of both backward compatibility and advanced functionality.

Please refer to the SB16 (Sound Blaster 16) sample to learn how to include and use the system-supplied DMusUART miniport driver in PortCls.sys.

DirectMusic kernel support is available in Windows 98 Second Edition and in Windows 2000 and later.
