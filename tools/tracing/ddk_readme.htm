Tracing Tools

The following software tracing tools are included in the Windows Driver Development Kit.

Instructions for using the tools are included in the "Tools for Software Tracing" section of the DDK Documentation. To find this section, in the documenation, go to Driver Development Kit, Tools, Tools for Software Tracing.

Trace Controllers
Trace controllers configure, start, stop, and update trace sessions and enable and disable trace providers.

·         Tracelog.exe
Controls real-time and trace log sessions from the command line. Controls all trace sessions running on the system.

·         Traceview.exe
Controls trace sessions and displays the trace messages from real-time tracing sessions and trace logs in a graphical user interface. You can also run TraceView from the command line. Controls only the trace session that it starts.

Trace Consumers
Trace consumers format and display trace messages, which are stored in binary format.

·         Tracefmt.exe
Formats trace messages from real-time trace sessions and trace log (.etl) files. Displays the message at the command line or writes them to human-readable output files.

·         Traceview.exe
Controls trace sessions and displays the trace messages from real-time tracing sessions and trace logs in a graphical user interface. You can also run TraceView from the command line. Controls only the trace session that it starts.

Other tracing tools

·         Tracepdb.exe
Creates trace message format (.tmf) files and trace message control (.tmc) files from PDB symbol files. Tracefmt and TraceView use the formatting instructions in the TMF files to format and display trace messages.
